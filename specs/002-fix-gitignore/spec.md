# Feature Spec: Fix .gitignore and Remove Tracked Build Artifacts

**Branch**: `002-fix-gitignore` | **Date**: 2026-03-07

## Problem

The repository tracks 6 compiled test binaries (`test_arena`, `test_counter`, `test_hello`, `test_layout`, `test_rdi_platform`, `test_vault`) that should be ignored by git. These are build artifacts generated by `make unit-tests` and `make integration-tests`, output to the repo root.

The `.gitignore` already covers `*.o` files (which are correctly untracked), but has no rule for the test binaries.

## Requirements

1. **R1**: Add `.gitignore` rules to exclude test binaries produced by the Makefile.
2. **R2**: Remove the 6 tracked test binaries from git's index without deleting them from disk.
3. **R3**: Ensure no other build artifacts are tracked or at risk of being tracked.
4. **R4**: Verify CI continues to pass after the change (test binaries are built fresh each run).

## Scope

- **In scope**: `.gitignore` update, `git rm --cached` for tracked binaries.
- **Out of scope**: Makefile refactoring, build directory restructuring (future work).

## Acceptance Criteria

- `git ls-files 'test_*'` returns empty after the fix.
- `make clean && make all && make unit-tests && make integration-tests` still passes.
- `.gitignore` covers all current and foreseeable build artifacts from the Makefile.
